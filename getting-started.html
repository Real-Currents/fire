Getting Started

Setting up all the depencies and a basic project in five easy steps.

1/5. Dependencies

Node on Fire has a few dependencies. Before you continue, make sure you have Node.js installed and have a Postgres development instance running. If you are on a Mac, you can use the Postgres app.

Now install Yeoman, Grunt, Bower and Node on Fire's Yeoman generator globally:

$ npm install -g yo grunt-cli bower generator-fire

There. That's it. Next step.

2/5. Database

Create a new development database locally and install the extension:

$ yo fire:datastore

  • Type the name of the database e.g. helloworld
  • Take note of the database url.

Feel free to change the names to whatever you want, just make sure to construct the correct database URL.

3/5. Creating the App

Scaffold the initial files and folders of the app:

$ yo fire:app

  • Type the name of the app e.g. hello-world
  • Paste the database URL from the previous step: postgres://fire:password@127.0.0.1/helloworld

The generator writes some files and installs the initial dependencies.

4/5. Creating the Controller

Now that we have an empty app, let's create a controller with a view which presents us the fabulous Hello, world!.

In index.js insert the following annotated piece of code before the app.start(); line:

app.template('index', '<div ng-controller="TestController"><h1>Hello, world!</h1></div>');

function TestController($scope, fire) {
	//
}
app.controller(TestController);

TestController.prototype.view = function() {
	return this.template('index');
};

5/5. Build, Release and Run

Build and release the project. This generates _assets, _api and _migrations in the build phase and migrates the database in the release phase:

$ grunt build && grunt release

$ node index.js

To view the app, head over to Hello, World!.
