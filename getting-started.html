Getting Started

Setting up all the depencies and a basic project in five easy steps.

1/5. Dependencies

Node on Fire has a few dependencies. Let's first make sure you have them installed.

  • Node.js & npm—http://nodejs.org/
  • PostgreSQL—http://postgresapp.com/ (Mac) or http://www.postgresql.org/download/
  • Yeoman—$ npm install -g yo or http://yeoman.io/
  • Grunt—$ npm install -g grunt-cli or http://gruntjs.com/
  • Bower—$ npm install -g bower or http://bower.io/
  • Node on Fire's Yeoman generator—$ npm install generator-fire

There. That's it. Next step.

2/5. Database

Set up a PostgreSQL database with a new user and install the neccesary extension:

  • Open psql (if using Postgresapp on Mac you can do so by clicking the elephant in the status bar and clicking Open psql)
  • Create the user—CREATE USER fire WITH PASSWORD 'password';
  • Create the database—CREATE DATABASE helloworld;
  • Grant privileges on the database to the user—GRANT ALL PRIVILEGES ON DATABASE helloworld TO fire;
  • Connect to the newly created database—\c helloworld
  • Install the UUID extension—CREATE EXTENSION "uuid-ossp";

The database is up and running. The URL to the database is postgres://fire:password@127.0.0.1/helloworld which we need in the next step.

Feel free to change the names to whatever you want, just make sure to construct the correct database URL.

3/5. Creating the App

Use the Yeoman generator we installed earlier to quickly scaffold an app:

$ yo fire

  • Type the name of the app e.g. hello-world
  • Paste the database URL from the previous step: postgres://fire:password@127.0.0.1/helloworld

The generator writes some files and installs the initial dependencies.

4/5. Creating the Controller

Now that we have an empty app, let's create a controller with a view which presents us the fabulous Hello, world!.

In index.js insert the following annotated piece of code before the app.run(); line:

app.template('index', '<div ng-controller="TestController"><h1>Hello, world!</h1></div>');

function TestController($scope, fire) {
	$scope.
}
app.controller(TestController);

TestController.prototype.view = function() {
	return this.template('index');
};

5/5. Build and Run

Before you can run the app, you need to build it. Node on Fire needs to generate some scripts before an app can run.

Now invoke $ grunt build:api && grunt build:scripts && grunt build:migrations which generates _assets, _api and _migrations.

A bug in grunt build.

It's currently not possible yet to simply call grunt build as there is a bug which creates invalid code when running all the build phases at once.

To run and view the app invoke $ node index.js and visit Hello, World!.
