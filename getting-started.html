Getting Started

Setting up all the depencies and a basic project in 4 easy steps.

1/4. Dependencies

Node on Fire has a few dependencies. Before you continue, make sure you have Node.js installed and have a Postgres development instance running. If you are on a Mac, you can use the Postgres app. Alternatively, you can use a free database from Heroku.

Now install Yeoman, Grunt, Bower and Node on Fire's Yeoman generator globally:

$ npm install -g yo grunt-cli bower generator-fire

2/4. Database

Create a new development database, if you don't have one yet, and install the "uuid-ossp" extension:

$ yo fire:datastore

  • Type the name of the database e.g. helloworld
  • Take note of the database url.

Feel free to change the names to whatever you want, just make sure to construct the correct database URL.

3/4. Creating the App

Scaffold the initial files and folders of the app:

$ yo fire:app

  • Type the name of the app e.g. helloworld
  • Paste the database URL from the previous step: postgres://fire:password@127.0.0.1/helloworld

The generator writes some files and installs the initial dependencies.

4/4. Ready to go!

You can now run the app:

$ cd helloworld/ (or whichever directory the project is in)

$ node index.js

To view the app, head over to http://127.0.0.1:3000/. When you make changes, run grunt build && grunt release again.
