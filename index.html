Node on Fire

A full-stack isomorphic web framework built on top of Node.js, AngularJS, Express and PostgreSQL.

Follow @nodeonfire

Embracing isomorphism

It's easy to create services which you can execute in both the client- and the server-side. Have a look at the isomorphic example project.

Express with dependency injection

Dependency injection is now also available on the back-end side, which makes creating Express routes super easy.

Integrated A/B testing

It's trivial to create A/B tests. You can A/B test every feature with ease. See A/B test example project.

Built on top of AngularJS, PostgreSQL, and more.

Node on Fire is built on top of existing technologies, like AngularJS, PostgreSQL, Grunt, Yeoman, RabbitMQ, and uses a collection of libraries, like Express and Knex.

The below list contains some more points you will like about Node on Fire.

  • An object-relational mapper to quickly build your data model and manage your database.
  • Migration-based data model changes which makes collaboration easy.
  • Flexible triggers module to create tasks to run after certain conditions e.g. send a user an email after 5 days of inactivity.
  • Easy workers module to invoke methods in the non-web process e.g. send a mail in a worker process whenver a new user signs up.
  • All of Node on Fire is Promise-aware. No more callback hell.
  • A framework following the 12factor methodology.
  • Easily deploy your app on Heroku or similar platforms.
  • Add multiple apps to one project e.g. your web app and the admin panel. See the multiple apps sample project.
  • Open-source. Everything is available on our repo.
