Node on Fire

A JavaScript-everywhere, model-driven, real-time web framework built on top of AngularJS, express, PostgreSQL and RabbitMQ.

What is Node on Fire?

Node on Fire is collection of JavaScript modules to rapidly create advanced web-worker applications following the 12factor methodology. Go to our getting started section to learn how to set up a working Hello World project.

Real-Time (Web Sockets)

Node on Fire supports web sockets and channels to push messages to and from clients.

function MyController(MyChannel, $scope) {
	var channel = MyChannel.get('test');
	channel.getMessage(function(message) {
		$scope.messages.push(messages);
	});

	channel.sendMessage({text: 'Hello, world!'});
}
app.controller(MyController);

Model-Driven

Configure your models in a non-awkward style:

function Article() {
	this.title	= [this.String, this.Required];
	this.url	= [this.String, this.Unique];
	this.createdAt 	= [this.DateTime];
}
app.model(Article);

Node on Fire automatically generates migrations, the database schema, a RESTful CRUD API and a client—server equal models API .

AngularJS

Node on Fire completely embraces the superheoric AngularJS framework for all things client-side.

function NewsController(ArticleModel, UserModel, $scope) {
	ArticleModel.find({}, {orderBy:{position:1}})
		.then(function(articles) {
			$scope.articles = articles;
		})

	$scope.voteArticle = function(article) {
		return article.createVoter({user: UserModel.getMe()});
	};
}
app.controller(NewsController);

Routing

Your controller methods are automatically routable and all views are automatically added to your $locationProvider. You don't have to keep track of separate routing tables for express and angular.

function TestController($scope) {
	$scope.test = 'Hello, world!';
}
app.controller(TestController);

TestController.prototype.view = function() {
	return this.template('index.jade');
};

The Process Model

In Node on Fire it's trivial to pass a message to a worker process. The below example sends an email in a worker process whenever a new user registers:

function User() {
	this.email = [this.String, this.Authenticate];
}
app.model(User);

User.prototype.afterCreate = function() {
	return this.workers.MailWorker.sendWelcomeMail(this);
}

function MailWorker() {

}
app.worker(MailWorker);

};

One-Off Tasks

You can create tasks which you can configure to run as scheduled task:

function DeactivateTask() {
	//
}
app.task(DeactivateTask);

DeactiveTask.prototype.run = function() {
	return this.models.User.update({active: true}, {active: false});
}

This works perfectly with Heroku's scheduler or any task scheduler as you can run them via the command line: $ node index.js --task DeactivateTask.
