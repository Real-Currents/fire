Node on Fire

A JavaScript-everywhere, model-driven, web framework built on top of AngularJS, express and PostgreSQL.

What is Node on Fire?

Node on Fire is a JavaScript-everywhere, model-driven, web framework. Instead of client- and server-side code, your code runs in the client-context, the server-context or both. Go to our getting started section to learn how to set up a working Hello World project.

JavaScript Everywhere

You write your complete app in JavaScript*, both the client- and server-side. You don't just put server code in /server/* and client code in /client/*, no, your controllers and models are truly available everywhere.

* If you prefer, you can also write you app in CoffeeScript, of course.

Model-Driven

Configure your models in a non-awkward style:

function Article() {
	this.title	= [this.String, this.Required];
	this.url	= [this.String, this.Unique];
	this.createdAt 	= [this.DateTime];
}
app.model(Article);

Node on Fire automatically generates migrations, the database schema, a RESTful CRUD API, client—server equal models API and client- and server-context routing.

AngularJS

Node on Fire completely embraces the superheoric AngularJS framework for all things client-side.

function NewsController(fire, $scope) {
	$scope.articles = fire.models.Article.find({}, {orderBy:{position:1}});
	$scope.user 	= fire.models.User.getMe();

	$scope.voteArticle = function(article) {
		return article.createVoter({user: $scope.user});
	};
}
app.controller(NewsController);

Scaffold Everything

Node on Fire loves Yeoman. A generator is available so you can scaffold your models, controllers, directives, or even the example apps. Install the generator via npm install -g generator-fire and run yo fire to get started with the generator.
